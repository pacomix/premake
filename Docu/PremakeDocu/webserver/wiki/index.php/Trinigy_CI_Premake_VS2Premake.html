What is VS2Premake?

  • VS2Premake extracts project's information from a Visual Studio 2008 project files (*.vcproj) and generates a project .lua script to be used in conjunction with the main Premake script.
  • It extracts the next information from a Visual Studio 2008 project file:
    • Project name.
    • Project GUID.
    • Preprocessor definitions.
    • Project build configs.
    • Include source directories.
    • Additional Library Paths.
    • Link Libraries.
    • Source file paths.
    • Files not using Precompiled Headers.
    • Excluded files.

How To Use?

  • The tool is located at trunk\vABC\Premake and the executable is VS2Premake.exe.
  • Execute the tool from the commandline, you can execute it from any directory if you have included trunk\vABC\Premake in your environment Path variable.
  • Defining a custom output name is optional, you need as minimum to define the .vcproj file to convert by using --file= (see below for description on parameters)
  • Parameters:
    • --file= - defines the relative or full path to the .vcproj file you want to convert.
    • --output= - defines the output name and path of the .lua script. 
Example:

 vs2premake --file="vbase90.vcproj" --output=../vBaseScript.lua

Remarks

  • This tool is only intended as a first step to aid in the generation of project script files. The result has to be always hand verified. For this look for the tag VS2PREMAKE inside the generated project script file.
  • While the converter has been tested and used for several projects, you always have to keep in mind it may output an incorrect value.

Known Issues

  • In some cases a source file reference may be placed inside sourcesNotUsingPCH{} file list. This results obviously in a build error and can be solved by manually moving the file to sourceFiles{}. This can be really tricky to detect in projects with a lot of files. The best approach to detect this problem is to open the .vcproj file and compare the file properties of the files included in sourcesNotUsingPCH against the original ones.
  • Using VS2Premake to extract the source and excluded file list of an already generated Visual Studio 2008 project file is not possible. The result is wrong. So it is not possible to work with Visual Studio adding and excluding files and then get again this list.
